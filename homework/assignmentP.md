# 数算B课程大作业说明及评分标准

*Updated 2026-04-26 16:27 GMT+8*  
 *Compiled by Hongfei Yan & Jialin Wang(2026 spring)*  



## 一、 作业目的
本大作业旨在鼓励同学们将课堂所学的计算机基础知识应用于解决实际问题。通过完成一个完整的项目，达到以下目标：
*   **工程实践：** 提升代码实现能力，掌握从需求分析到项目交付的全流程。
*   **深度内化：** 在实战中理解算法的边界条件、时空复杂度及适用场景。
*   **创新表达：** 结合个人兴趣（如游戏、工具、数学建模、AI应用等），展现跨学科应用能力。
*   **社区参与：** 熟悉 GitHub 开发工作流，培养开源精神与文档规范意识。

## 二、 作业要求

### 1. 选题方向（建议但不限于）
*   **经典游戏复刻与AI：** 如棋类游戏（五子棋、六子棋）、迷宫生成与求解、24点算法求解器等。建议包含非平庸的算法逻辑（如 Minimax 算法、Alpha-Beta 剪枝、回溯法等）。
*   **算法可视化工具：** 动态展示经典算法（如 最短路径寻路、各种排序过程、树/图的遍历等）的执行流程，辅助他人理解算法。
*   **算法应用工具：** 
    *   **数论工具：** 大数/矩阵运算库、素性检验与因数分解工具、RSA 加解密演示。
    *   **文本处理：** 基于 KMP 的匹配工具、Huffman 压缩解压器、简易 Markdown 解析器。
    *   **数据分析：** 结合复杂数据结构的成绩分析、关系网络图分析。
*   **自主构思：** 任何能体现“数据结构”、“算法”或"人工智能"核心价值的项目。

### 2. 技术与工程要求
*   **编程语言：** 不限（推荐 C++/Python/Java/Rust/JavaScript 等）。
*   **工程质量：** 代码需模块化，逻辑与界面分离（涉及 GUI 的项目），无死循环或严重内存泄漏。
*   **环境说明：** 需提供明确的运行环境说明（如 requirements.txt、Makefile 或环境配置步骤）。

### 3. 提交规范（含命名规范）

请通过 Canvas 系统提交：

* **<mark>提交内容</mark>：**

  1. **PDF 主文档：** 包含项目 GitHub 仓库链接、项目核心截图、以及 README 的内容预览。
  2. **附件补充：** 请将原始的 .md 或 .doc 格式文档，作为附件上传至提交页右侧的**“作业评论”**区（方便快速查阅）。

*   **GitHub 仓库命名规范：**
    
    *   格式：`[项目英文简称]`
    *   示例：`SudokuSolver`、`GraphVisualizer`、`RSA-Toolkit`。
    
*   **仓库文件结构建议：**
    
    ```text
    ├── src/            # 核心源代码
    ├── docs/           # 文档、演示素材、报告等
    ├── tests/          # 测试用例或测试脚本
    ├── README.md       # 项目主说明文档（必选）
    ├── LICENSE         # 开源协议（如 MIT）
    └── requirements.txt # 环境依赖清单
    ```
    
*   **README 文档要求：**
    
    *   **项目背景：** 说明选题初衷及解决的问题。
    *   **核心算法：** 详细说明使用了哪些课程知识点（如：使用了 AVL 树优化查询、应用了动态规划求解最优路径等）。
    *   **运行指南：** 清晰的操作步骤，确保能顺利运行。。
    *   **AI 工具声明：** 若使用了 AI（如 Gemini, ChatGPT, Copilot 等），请简要说明 AI 辅助的部分（如“利用 AI 生成了 GUI 框架代码，核心回溯算法逻辑由本人独立实现”）。

### 4. 学术诚信声明
*   **严禁抄袭：** 允许引用开源代码或参考 AI 生成的代码，但必须在 README 中**显著标出**参考来源。项目的核心逻辑与系统架构应体现作者的自主思考。

## 三、 评分标准（总分 100 分）

| 维度           | 分值 | 优（90-100）                                                 | 良（75-89）                                  | 及格（60-74）                                                |
| :------------- | :--- | :----------------------------------------------------------- | :------------------------------------------- | :----------------------------------------------------------- |
| **核心功能**   | 30   | 功能完整，运行流畅，鲁棒性强，完全达成甚至超过预期。         | 功能基本完整，存在极个别边缘 Case 下的 Bug。 | 核心功能有缺失，或运行不够稳定。                             |
| **知识点应用** | 30   | **深度应用**课程课程知识点，且对算法选型有合理的分析与自我实现。 | 能够正确应用课程知识点，逻辑准确。           | 知识点应用肤浅，大量依赖外部库函数而缺乏底层实现。           |
| **代码质量**   | 20   | 命名规范，注释清晰，结构优雅，代码解耦良好。                 | 逻辑清晰，有基本注释，代码可读性较好。       | 代码风格杂乱，存在硬编码或大量逻辑冗余。                     |
| **文档与工程** | 10   | README 专业且美观，含截图或演示，环境配置简单明了。          | 文档要素齐全，运行指南清晰。                 | 文档简陋，需花费额外精力才能跑通项目。                       |
| **创新性**     | 10   | 选题独特，或在算法效率、交互设计上有显著亮点。               | 有自主思考，非简单的课堂作业延伸。           | 达到<mark>基本要求</mark>（如在 GitHub 维护了系统的课程学习总结）。 |
